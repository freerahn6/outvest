# 종목노트 집필·검수 규약

> 목적: **글이 나가기 전에, 작성자가 아닌 다른 애널리스트가 오류를 잡는다.**
> 가장 흔한 사고는 수치·날짜 오류다(실제로 시황 글의 잠정치를 다음날 정정한 사례가 있다).
> 이 규약은 그 오류를 *배포 후 정정*이 아니라 *배포 전 차단*으로 옮기기 위한 것이다.
>
> 이 문서는 회사 PC·집 PC 공통 규약이다(레포와 함께 동기화된다). 감마는 주식블로그 글을
> 쓰기 전에 이 문서를 참조하고, 이 절차대로 오케스트레이션한다.

---

## 1. 원칙

1. **작성자는 자기 글을 검수하지 않는다.** 검수는 반드시 다른 애널리스트가 한다.
2. **검수는 복수 관점이다.** 최소 (a) 사실성 팩트체크 1명 + (b) 리스크·면책 검수(an7 상시) 1명.
3. **글 성격에 따라 검수 강도를 달리한다.** 수치·날짜가 많은 글일수록 강하게 본다(§3).
4. **미해결 지적은 감마가 판정한다.** 작성자와 검수자가 갈리면 감마가 결정하고, 결정 근거를 남긴다.
5. **면책 문구 없는 글은 배포하지 않는다.**

---

## 2. 역할 배정

### 작성자(1명) — 주제로 배정
| 글 주제 | 1순위 작성자 |
|---|---|
| 시황·거시·자금흐름 | an1(종합) / an2(거시·지정학) |
| 기술적 분석·차트·패턴 | an3(정통 차트) / an4(수급·변동성·퀀트) |
| 산업·기술(반도체·배터리·AI) | an5 |
| 재무·밸류에이션·실적 | an6 |
| 리스크·포트폴리오·투자심리 | an7 |
| 기초강의(개념 설명) | 주제 인접 담당 |

### 검수자(작성자 제외, 2명 이상)
- **팩트체커** — 주제에 인접한 전문가를 배정한다.
  - 밸류에이션·재무 수치 → **an6**
  - 지수·금리·환율·거시 수치 → **an2**
  - 수급·거래량·차트 레벨 → **an4**
  - 산업 사실관계(점유율·로드맵·밸류체인) → **an5**
- **리스크·면책 검수 = an7 상시.** an7은 블로그 신뢰도·면책 톤의 균형추다. 모든 글에 붙는다.
  - (an7이 작성자인 글은 리스크·면책 검수를 an1이 대신 맡는다.)

> 배정 원칙: 팩트체커는 **그 수치를 실제로 계산·검증할 수 있는 사람**이어야 한다.
> "읽어봤다"가 아니라 "다시 계산/대조했다"가 검수다.

---

## 3. 글 성격별 검수 강도

| 등급 | 대상 | 검수 |
|---|---|---|
| **강(强)** | 시황·데이터 중심 글(지수·등락률·수급 금액·실적 수치·기준일이 많은 글) | 팩트체커 + an7 **필수**, 수치 전수 대조 |
| **표준** | 개별 종목·산업 분석 | 팩트체커 + an7, 핵심 수치·주장 대조 |
| **경량** | 기초강의·에버그린(개념 설명, 실시간 수치 없음) | 검수 1명(개념 정확성·면책·오해소지) |

판별 질문: **이 글에 "틀리면 다음날 정정해야 할 수치·날짜"가 있는가?** → 있으면 강/표준, 없으면 경량.

---

## 4. 절차

```
① 감마: 주제 → 작성자 배정 + 검수 강도 판정(§3)
② 작성자: 초안 작성 (프론트매터 draft: true 로 시작)
③ 검수: 팩트체커 + an7(또는 §3 경량은 1명)이 각각 검수 리포트 제출
④ 작성자: 지적 반영 또는 반론
⑤ 감마: 미해결 지적 판정 → 통과 시 draft 해제 → 커밋 → git push origin main (배포)
```

- 초안은 **`draft: true`** 로 두면 사이트에 노출되지 않는다(README 2절). 검수 통과 후 해제한다.
- 검수 리포트는 대화(오케스트레이션) 안에서 주고받으며, 남길 가치가 있으면 커밋 메시지에 요지를 적는다.

## 5. 검수 체크리스트

검수자는 아래를 **항목별로 통과/지적**으로 답한다("대충 괜찮음" 금지).

### A. 사실성 (오류의 90%가 여기서 난다)
- [ ] 모든 수치(지수·등락률·금액·PER·점유율 등)에 **출처와 기준일**이 있는가?
- [ ] **잠정치는 "잠정치"로** 명시했는가? (수급 등 매체 간 편차 있는 값)
- [ ] 날짜·종목명·**종목코드·티커**가 정확한가? (프론트매터 `tickers` 코드 포함)
- [ ] 인용한 통계·발언·사건이 **실재하는가**(창작·추정을 사실로 쓰지 않았는가)?
- [ ] 계산이 필요한 수치를 **다시 계산해 대조**했는가?

### B. 논리·톤
- [ ] 예측·의견을 **사실처럼 단정**하지 않았는가?
- [ ] 주장이 **반증가능한가** — 틀렸다면 무엇으로 드러나는지 글이 말하는가?
- [ ] 과장·낚시 없이 **근거가 결론을 지지**하는가?
- [ ] 앞 글과 모순되면 그 사실을 **정직하게 밝혔는가**?

### C. 규정·신뢰도
- [ ] **투자 면책 문구**가 있는가?
- [ ] 특정 종목 **매수·매도 권유**로 읽히지 않는가?
- [ ] 링크·이미지(커버) 경로가 실제로 존재하는가?

> 지적 심각도: **[치명]**(수치·사실 오류, 면책 누락) → 반드시 수정 / **[권고]**(톤·표현) → 작성자 재량.
> [치명] 미해결 상태로는 배포하지 않는다.

---

## 6. 이 규약의 근거

- 애널리스트 창단 훈련 R1(2026-07-11) 공통 급소:
  **C-1 "주장(claimed) ≠ 실제(actual)"** — 자기평가는 검증이 아니라 알리바이가 되기 쉽다 → 그래서 **타인 검수**가 필요하다.
  **C-2 "반증지표는 경쟁 가설을 겨눠야"** → 체크리스트 B의 반증가능성 항목.
- 실제 사고: 시황 글의 1순위 지표·수급 수치가 틀려 다음날 정정 → 배포 전 팩트체크로 옮긴다.
